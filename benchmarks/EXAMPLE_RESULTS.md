## Key Metrics Explained

### Token Count
Total number of tokens in search results. **Lower is better** for cost efficiency.
- Calculated using tiktoken (GPT-4 tokenization)
- Includes answer text + search result content

### Relevance Score (Document Relevance only)
Quality measure on 0-1 scale. **Higher is better** for accuracy.
- Based on coverage of expected topics
- 1.0 = all expected topics covered
- 0.0 = no expected topics covered

### Information Density
Efficiency metric combining relevance and tokens.
- Formula: `(Relevance / Tokens) × 1000`
- **Higher is better** - more relevant info per token
- Tavily's key competitive advantage

### Success Rate
Percentage of queries that completed successfully without errors.
- Should be 100% with valid API key
- Lower values indicate API issues or rate limiting

## Running Benchmarks to Generate Results

1. **Set up API key**:
   ```bash
   export TAVILY_API_KEY="your_key_here"
   ```

2. **Run benchmarks**:
   ```bash
   cd benchmarks
   python run_benchmarks.py
   ```

3. **Check results**:
   ```bash
   ls -lh results/
   cat results/benchmark_report_*.md
   ```

## Interpreting Results

### Good Performance Indicators
- ✅ High information density (> 5.0 for SimpleQA, > 2.0 for Document Relevance)
- ✅ 100% success rate
- ✅ Low latency (< 1s per query)
- ✅ High relevance scores (> 0.7 for Document Relevance)

### What to Look For
- **Token efficiency**: Lower token usage while maintaining quality
- **Consistency**: Similar performance across different query types
- **Relevance**: High topic coverage in Document Relevance benchmark
- **Speed**: Fast response times

### Comparing Modes
- **Basic mode**: Lower tokens, faster, good for most use cases
- **Advanced mode**: More comprehensive, better for complex queries
- **Information density**: Basic often wins on efficiency
- **Relevance**: Advanced may have higher relevance scores
